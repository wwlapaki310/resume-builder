# Google シニアソフトウェアエンジニア（生成AI）応募ガイド

## このポジションの重要ポイント

### 1. テクニカルソリューションアーキテクトの役割
このポジションは単なるソフトウェアエンジニアではなく、**顧客対応を伴うソリューションアーキテクト**の側面が強いです。

**重視されるスキル：**
- 顧客の技術的課題を見極める能力
- ソリューション提案・アーキテクチャ設計
- デモ・概念実証（PoC）の作成
- プレゼンテーション・説明力

### 2. 生成AI（GenAI）の専門性
Google Cloud Platform の生成AIチームという特殊性があります。

**求められる知識：**
- 大規模言語モデル（LLM）の理解
- Generative Language API の知識
- AI専用インフラストラクチャ（TPU、GPU）
- 責任あるAIの実践

### 3. バイリンガル必須
日英両言語での**ビジネスレベル**のコミュニケーションが必須です。

**確認ポイント：**
- 技術的内容を英語で説明できるか
- 日本語で顧客サポートができるか
- 両言語でのプレゼンテーション経験

## 応募書類作成のベストプラクティス

### Resume（職務経歴書）

#### ✅ DO（すべきこと）
1. **数値で成果を示す**
   - "パフォーマンスを改善" → "推論速度を40%向上"
   - "コストを削減" → "インフラコストを月$5,000削減"
   - "ユーザー体験を改善" → "応答時間を2秒から0.5秒に短縮"

2. **求人要件に合わせた経験を強調**
   - ML/AI: 5年以上 → 具体的な年数と技術を明記
   - ソフトウェア開発: 5年以上 → 言語と開発規模を記載
   - アーキテクト経験 → 顧客対応、PoC作成の実績

3. **Google Cloud / GCP の経験をアピール**
   - Vertex AI、BigQuery、Cloud Run等の使用経験
   - GCP上でのMLシステム構築
   - Google Cloud認定資格

4. **生成AI関連の実績**
   - LLM を使ったアプリケーション開発
   - RAG、Fine-tuning、Prompt Engineering
   - OpenAI API、Anthropic Claude、Gemini等の経験

#### ❌ DON'T（避けるべきこと）
1. 曖昧な表現
   - "様々な" → 具体的な技術名を列挙
   - "多くの" → 具体的な数値
   - "貢献した" → 自分の具体的な役割

2. 職務内容の羅列
   - 業務内容だけでなく「成果」を記載
   - What だけでなく Impact を示す

3. 無関係な経験の詳述
   - Google Cloud AIに関連性の低い経験は簡潔に

---

### Cover Letter（カバーレター）

#### 構成の鉄則
1. **Opening**: なぜこのポジションか（2-3文）
2. **Body 1**: ML/AI の技術的専門性
3. **Body 2**: ソリューションアーキテクト・顧客対応
4. **Body 3**: バイリンガル能力、コミュニケーション
5. **Closing**: 貢献できる価値、面接への期待

#### トーンとスタイル
- **プロフェッショナル**だが**熱意**が伝わる
- **自信**を示すが**謙虚さ**も保つ
- **具体的**な実績を盛り込む

#### 長さ
- 英語: 300-400語
- 日本語: 600-800字
- A4用紙1枚程度

## 面接準備（書類通過後）

### 想定される質問

**技術面接**
- ML/AIアルゴリズムの説明
- システム設計（スケーラビリティ、パフォーマンス）
- 過去のプロジェクトでの技術的課題と解決策
- コーディング課題（Python/Java/C++）

**行動面接**
- 顧客の難しい技術的課題をどう解決したか
- チーム内での対立をどう解決したか
- 期限内に成果物を届けた経験
- 失敗から学んだこと

**ケース面接**
- 顧客シナリオに基づくソリューション提案
- 生成AIを使った新しいユースケースの提案
- アーキテクチャ設計の実演
